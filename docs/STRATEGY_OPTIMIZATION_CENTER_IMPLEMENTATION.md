# 策略优化中心实现说明

## 1. 目的

策略优化中心不是普通采购处理页面，而是给管理员、算法负责人和供应链专家使用的调参和回测页面。

业务页面负责处理今天的建议：

- 缺料风险
- 采购资金改善
- 主数据体检
- 审批包
- SAP 回写草稿

策略优化中心负责决定系统以后如何生成建议：

- ABC/XYZ 服务水平
- 缺料风险告警阈值
- 冻结期
- 审批金额阈值
- 风险惩罚和供应商扰动惩罚
- 策略版本保存、回测和人工确认

## 2. 已实现内容

### 2.1 策略预设

代码：`chainpilot_ai/strategy/policy.py`

内置三组策略：

| 策略 | 定位 | 特点 |
| --- | --- | --- |
| 保守策略 | 降低缺料漏报 | 更高服务水平、更低告警阈值、更长冻结期 |
| 均衡策略 | 默认试点 | 风险和资金改善平衡 |
| 进取策略 | 提高资金改善 | 更短冻结期、更高资金动作上限 |

所有策略都强制：

```text
sap_writeback_mode = draft_only
```

系统不会自动写 SAP。

### 2.2 Mock 历史快照

代码：`chainpilot_ai/strategy/mock_history.py`

Mock 阶段已经不是静态样例，而是生成可回放历史：

- 生成多期 SAP 历史快照。
- 按日期平移 PR、PO、需求、收货和供应商履约记录。
- 对高风险物料增加需求和供应延迟。
- 对高库存物料模拟过量库存和可释放资金。
- 每个快照生成 `actual_outcomes`，用于验证预测是否命中真实缺料。

后续接真实 SAP 两年历史数据时，替换这个快照生成器，不重写回测和页面。

### 2.3 生产级算法接线

已把策略参数接入算法运行：

- 缺料风险算法读取 ABC/XYZ 分层阈值和服务水平。
- 采购资金改善算法读取冻结期、最低库存天数、审批金额阈值、最大动作数、风险惩罚、供应商扰动惩罚。
- 回测运行三类算法：缺料风险、采购资金改善、主数据诊断。

当前优化器仍保留可解释的排序求解，后续可把 `_select_actions` 替换为 OR-Tools/HiGHS；输入输出契约已经按策略和硬约束组织。

### 2.4 回测指标

代码：`chainpilot_ai/strategy/backtest.py`

每个策略输出：

| 指标 | 含义 |
| --- | --- |
| 缺料召回率 | 实际缺料中被提前识别的比例 |
| 高风险准确率 | 预测高风险中真实缺料的比例 |
| 预计资金改善 | 策略建议动作的资金改善 |
| 预计兑现资金 | 考虑供应商确认和审批后的模拟兑现 |
| 误干预率 | 预测高风险但没有实际缺料的比例 |
| 硬约束违规 | 被选中动作违反硬约束的数量 |
| 策略评分 | 风险、收益、误干预和硬约束的综合评分 |

策略启用前必须人工确认。

## 3. 页面

页面：`strategy-optimization-center`

入口：

```text
http://chainpilot.localhost:8000/app/strategy-optimization-center
```

页面内容：

1. 顶部指标：推荐策略、历史窗口、回写方式、上线控制。
2. KPI：缺料召回率、高风险准确率、预计兑现资金、策略评分。
3. 策略对比：保守、均衡、进取三组策略。
4. 参数配置：ABC/XYZ 服务水平和缺料告警阈值。
5. 回测样本：命中、误报、资金动作等明细。
6. 上线闸门：硬约束、人工确认、策略版本、真实数据状态。

## 4. DocType

新增：

- `Decision Strategy`
- `Backtest Run`
- `Backtest Result`

点击页面“运行回测”或策略卡片“保存”后，回测结果会落库，便于后续审批和追溯。

## 5. 后续接真实数据

真实 SAP 历史数据接入后按以下方式替换：

1. 用 OData/RFC/文件导入生成每日历史 Snapshot。
2. 保留和 mock 相同的数据契约。
3. 用前 18 个月做策略校准。
4. 用后 6 个月做留出验证。
5. 保守、均衡、进取策略仍由系统推荐，人确认后启用。
