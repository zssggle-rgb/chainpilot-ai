# 模拟账套工作台界面设计

## 1. 参考方向

主参考为 Anaplan Supply Planning。该类计划工作台强调：

- 以计划场景为入口，支持基准场景、优化场景和参数调整。
- 以供需平衡、库存策略、订单短缺和容量/约束异常组织页面。
- 首屏展示计划员需要立即判断的指标，而不是数据快照编号、同步时间等技术信息。
- 主体使用图表加表格的混合工作区，用户可以从汇总指标下钻到 SKU/物料、地点、供应商和订单行。

参考资料：

- Anaplan Supply Planning：可视化供应链，从库存、订单短缺和容量异常管理供应计划。
- Anaplan Capacity Planning：左侧有计划流程和运行优化入口，主体展示周度容量/需求图和明细表。
- Anaplan Inventory Planning：围绕库存策略、服务水平、需求波动和库存覆盖组织分析。

## 2. ChainPilot 页面定位

页面名称：`供应计划验证工作台`

目标用户：

- 采购计划员：看哪些物料缺料、哪些 PR/PO 可以调整。
- 供应链经理：看资金占用、缺料风险、供应商履约和策略效果。
- 项目演示对象：判断模拟数据是否真实、算法是否基于可信明细运行。

页面不再定位为“数据中心”。它应展示一套可运行的计划场景：

- 有真实感的物料、库存、需求、采购申请、采购订单、供应商履约和计划参数。
- 能看到供需缺口、库存策略、采购建议和约束阻断。
- 能看到算法效果和回测指标。
- 技术快照信息只作为审计内容，不进入首屏。

## 3. 信息架构

### 3.1 左侧场景栏

固定展示当前计划场景：

- 场景名称：基准模拟账套。
- 计划周期：未来 45 天。
- 工厂范围：CN01、CN02。
- 产品范围：空调、商显、通用。
- 操作按钮：运行优化、配置 SAP 接入、查看回测。

### 3.2 首屏 KPI

只展示业务指标：

- 缺料风险物料。
- 预计减少资金占用。
- 采购未清金额。
- 供应履约稳定性。

不在首屏展示：

- 快照编号。
- 快照时间。
- 后端方法名。
- 算法内部字段。

### 3.3 主体 Tab

按计划员工作流组织：

1. `供需平衡`
   - 周度需求、已确认供给、计划供给、净缺口。
   - 高风险物料队列。

2. `库存策略`
   - 物料分层、库存覆盖、安全库存、采购未清、缺料概率。
   - 库存策略建议，如提高安全库存、维持策略、降低目标库存。

3. `采购计划`
   - 采购申请和采购订单明细。
   - 显示单据行、物料、供应商、数量、交期、未清金额、确认状态。

4. `供应商履约`
   - 供应商历史交付、延迟天数、质量异常。
   - 用于支撑供应风险判断。

5. `优化建议`
   - 已入选的 PR/PO 调整动作。
   - 缺料风险清单。
   - 不可处理原因分布。

6. `数据校验`
   - 跨表关联校验。
   - 约束覆盖。
   - 技术快照与求解器信息。

## 4. 视觉系统

整体风格：

- 低饱和企业软件风格。
- 更密集的信息布局。
- 少用大面积解释文案。
- 图表和表格优先，卡片只承载关键指标或对象。

色彩：

- 主色：深蓝灰，用于导航、标题和主按钮。
- 强调色：蓝色用于当前场景和选中状态。
- 成功色：绿色用于可执行、通过、资金改善。
- 风险色：琥珀色和红色用于缺料、延迟、阻断。

组件：

- 左侧场景栏。
- 顶部 KPI 条。
- 周度供需条形图。
- 物料工作表。
- 状态标签。
- 紧凑业务表格。

## 5. 文案原则

- 不显示“本页面用于”“根据什么计算”这类过程说明。
- 不显示英文或技术缩写，除非是业务系统通用对象且中文解释明确。
- 不用“AI 推荐”“智能洞察”等空泛表达。
- 每个标题必须对应一个业务对象或业务动作。

示例：

- 使用：`缺料风险物料`
- 不使用：`AI 识别结果`
- 使用：`预计减少资金占用`
- 不使用：`现金优化输出`
- 使用：`采购申请`
- 不使用：`PR 行项目数据源`

## 6. 验收标准

- 首屏不出现快照编号、快照时间。
- 页面必须包含供需平衡、库存策略、采购计划、供应商履约、优化建议、数据校验六个业务页签。
- 页面必须能展示物料、库存、需求、采购申请、采购订单、供应商履约等明细。
- 页面必须显示周度供需关系，而不是只显示数据行数。
- 页面必须显示算法建议与阻断原因。
- 技术信息必须放在数据校验页签下。
