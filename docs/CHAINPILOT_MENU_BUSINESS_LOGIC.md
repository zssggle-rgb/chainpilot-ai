# ChainPilot AI 左侧菜单业务逻辑

本文说明左侧菜单的业务闭环、每个页面的使用方式，以及演示时应该观察哪些结果。当前系统使用 SAP 模拟快照跑通完整流程；接入真实 SAP 后，数据来源从模拟快照切换为只读同步的 SAP 历史与当前数据，页面业务逻辑保持不变。

## 业务闭环

ChainPilot AI 的左侧菜单不是按技术模块组织，而是按供应链计划员每天的处理顺序组织。

1. 数据准备：从“数据中心”和“数据连接”确认物料、库存、需求、采购申请、采购订单、供应商和主数据参数是否完整。
2. 策略验证：在“优化训练”和“计划场景”回放历史数据、比较约束方案、确认参数是否适合当前业务。
3. 日常处理：在“计划工作台”查看推荐清单，然后进入“缺料例外”“采购动作”“主数据”处理具体对象。
4. 审批执行：通过“动作队列”“对象详情”“执行监控”完成审批、回写草稿和兑现跟踪。
5. 反馈学习：在“学习中心”和“智能协同”沉淀拒绝原因、供应商反馈、兑现结果和业务目标。

## 页面说明

### 计划工作台

入口：`/app/chainpilot-ai-command-center`

业务目的：计划员每天进入的首页。它回答三个问题：今天哪些物料有缺料风险，哪些采购申请或采购订单可以调整，哪些主数据参数需要复核。

页面结构：使用“推荐处理 / 对象队列 / 使用流程”三个页签。“推荐处理”是核心清单，展示 SAP 对象、物料、工厂、供应商、推荐动作、当前值、建议值、资金影响、风险和约束状态。

怎么用：先按风险和资金影响看推荐清单，点击行项目查看右侧证据，再打开对象详情或生成审批包。

怎么看效果：不是看 KPI 数字，而是看每条建议是否有 SAP 单据、算法运行、证据、约束和下一步动作。

### 缺料例外

入口：`/app/shortage-risk-war-room`

业务目的：处理未来 14 天可能影响生产的缺料物料。

页面结构：页签包括“风险物料 / 库存投影 / 处置动作”。风险物料列表给出预计缺料日、P90 缺口、缺料概率、影响工单和建议动作；库存投影展示当前库存、预计到货、需求、安全库存线和缺口。

怎么用：选择风险物料，核对库存投影和影响工单，决定催交采购订单、启用替代料或新建紧急采购申请。

怎么看效果：看高风险物料是否被识别出来，建议动作是否能降低缺料概率，且不违反冻结期、MOQ、供应商交期等约束。

### 采购动作

入口：`/app/cash-release-action-package`

业务目的：把可减少采购资金占用的 PR/PO 调整动作整理成审批包。

页面结构：页签包括“目标约束 / 低风险可审批 / 供应商确认 / 被阻断”。表格展示 SAP 对象、物料、原值、建议值、资金影响、风险变化和约束状态。

怎么用：先看目标约束，再处理低风险动作；需要供应商确认的动作进入采购沟通，被阻断动作只保留证据，不进入审批包。

怎么看效果：看现金改善金额是否来自具体 PR/PO 行项目，且每条动作都有服务水平、冻结期、MOQ/MPQ、安全库存等约束结论。

### 主数据

入口：`/app/master-data-health`

业务目的：识别会导致 MRP 结果失真的 SAP 主数据参数。

页面结构：页签包括“健康概览 / 计划交货期 / 安全库存 / MOQ/MPQ / 供应商参数”。每条异常展示当前 SAP 值、建议值、样本数、P80/P95、置信度和影响金额。

怎么用：按参数类型复核异常，把确认后的参数变更进入主数据修正包。

怎么看效果：看建议值是否来自历史样本分布和供应商履约证据，而不是单纯人工拍数。

### 数据中心

入口：`/app/mock-data-center`

业务目的：展示当前模拟账套到底包含哪些业务对象，证明算法不是只看汇总数。

页面结构：页面本身已有对象页签，用于查看物料、库存、需求、采购申请、采购订单、供应商、BOM/替代料、主数据参数和历史样本。

怎么用：在演示前先打开数据中心，说明模拟数据覆盖了真实 SAP 计划场景中的关键对象和关联关系。

怎么看效果：看物料、工厂、供应商、PR、PO、需求、库存、主数据是否能互相串起来；如果缺少关联，算法推荐就没有说服力。

### 数据连接

入口：`/app/sap-integration-console`

业务目的：配置真实 SAP 只读接入和字段映射。

页面结构：页签包括连接配置、接入参数、接口配置、同步任务和接口日志。需要配置连接方式、租户或系统地址、客户端、认证方式、实体集、字段映射、增量水位和目标快照表。

怎么用：先配置连接和认证，再为物料、库存、需求、PR、PO、供应商和主数据对象配置接口；校验通过后执行只读同步。

怎么看效果：看同步任务是否有对象级数量、错误摘要、字段映射结果和目标快照覆盖情况。

### 优化训练

入口：`/app/strategy-optimization-center`

业务目的：在模拟或历史数据上回测算法策略，决定哪些参数可以进入生产试点。

页面结构：页签展示策略对比、参数配置、回测样本和上线闸门。

怎么用：选择策略版本，查看历史回放结果，再按服务水平、缺料率、现金改善和审批通过率判断是否上线。

怎么看效果：看策略不是只在一批样本上表现好，而是在多个历史窗口中都能稳定减少风险和资金占用。

### 计划场景

入口：`/app/scenario-studio`

业务目的：让业务人员用目标和约束构造方案，然后比较多个计划方案。

页面结构：页签包括业务目标、SAP 快照和方案对比。

怎么用：输入现金释放目标、服务水平要求、保护品类和禁止动作，生成保守、推荐、进取等方案。

怎么看效果：看不同方案的收益、风险、建议数量和约束冲突是否清晰，便于业务选择。

### 动作队列

入口：`/app/action-inbox`

业务目的：集中处理待审批、待沟通、待复核的动作。

页面结构：页签包括“待审批动作 / 缺料风险处理 / 主数据复核”。

怎么用：按队列处理动作，进入采购动作、缺料例外或主数据页面查看完整证据。

怎么看效果：看动作是否能从队列追溯回 SAP 对象和算法证据，而不是独立任务卡片。

### 执行监控

入口：`/app/execution-monitor`

业务目的：跟踪审批包、SAP 回写草稿、供应商沟通和实际兑现。

页面结构：页签包括审批任务、SAP 回写草稿、供应商沟通、执行结果和学习信号。

怎么用：审批通过后生成回写草稿，人工确认后再进入真实 SAP；供应商沟通和执行结果回流学习中心。

怎么看效果：看建议是否真正落地，例如 PR 数量是否下调、PO 是否延后、现金改善是否兑现、是否引入缺料风险。

### 学习中心

入口：`/app/learning-center`

业务目的：把采纳、拒绝、供应商反馈、兑现结果和缺料事件沉淀成后续排序和阈值调整依据。

页面结构：页签包括学习洞察、指标快照、规则调权草稿、拒绝原因、供应商反馈、兑现结果、缺料事件和学习信号。

怎么用：复盘每轮建议的采纳率、拒绝原因和兑现结果，再决定是否调整策略权重。

怎么看效果：看学习信号是否能解释为什么某类动作应该升权、降权或补充证据。

### 智能协同

入口：`/app/ai-copilot`

业务目的：把业务目标转成可执行的计划场景、审批摘要和供应商沟通材料。

页面结构：页签包括业务目标、最近输出、运行历史、处理记录和数据质量提示。

怎么用：输入业务目标，例如“在不降低关键物料服务水平的前提下减少采购占用”，系统生成方案约束、解释材料和沟通草稿。

怎么看效果：看它是否围绕具体 SAP 对象和算法结果生成材料；不能只给泛泛文字。

### 对象详情

入口：`/app/recommendation-detail`

业务目的：审查单条推荐动作是否可以审批和执行。

页面结构：页签包括“当前建议 / 算法追踪 / 证据链 / 约束校验 / 执行时间线”。

怎么用：按 Before/After 核对调整内容，再看算法、证据和约束，最后提交审批或生成回写草稿。

怎么看效果：看每条动作能否完整追溯：SAP 对象 -> 算法运行 -> 证据 -> 约束 -> 审批 -> 回写草稿。

### 优化运行

入口：`/app/algorithm-run-detail`

业务目的：查看算法每次运行的输入、输出、质量指标和关联对象。

页面结构：页签包括“运行列表 / 关联 SAP 对象”。运行列表展示预测误差、最优差距、结果数量和耗时。

怎么用：在回测或演示时打开运行详情，说明推荐清单来自哪次算法运行。

怎么看效果：看算法输出是否稳定，是否存在不可行、截断、超时或质量不足的运行。
